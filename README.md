# UNMSM - ISW - Entornos Metodológicos de Desarrollo de Software

## 📚 Descripción del Proyecto

Este repositorio alberga los recursos, materiales de estudio, ejemplos de código, y proyectos prácticos relacionados con el curso de **Entornos Metodológicos de Desarrollo de Software** de la Escuela Profesional de Ingeniería de Software de la Universidad Nacional Mayor de San Marcos (UNMSM).

El objetivo principal de este curso es proporcionar a los estudiantes una comprensión sólida y práctica de las metodologías ágiles, herramientas y prácticas modernas utilizadas en el ciclo de vida del desarrollo de software. Se busca fomentar la colaboración, la eficiencia y la calidad en la construcción de sistemas de software robustos y escalables.

## ✨ Objetivos del Curso

* Comprender y aplicar los principios y valores de las metodologías ágiles (Scrum, Kanban, etc.).
* Aprender a utilizar herramientas de gestión de proyectos y control de versiones (Git, GitHub/GitLab).
* Explorar entornos de integración continua (CI/CD) y despliegue continuo.
* Familiarizarse con prácticas de desarrollo de software como TDD (Test-Driven Development), BDD (Behavior-Driven Development) y refactoring.
* Desarrollar habilidades para trabajar en equipos colaborativos en proyectos de software.
* Aplicar patrones de diseño y arquitecturas de software modernas, incluyendo microservicios.
* Integrar soluciones de seguridad como Single Sign-On (SSO) y gestión de identidades.

## 📂 Estructura del Repositorio

El repositorio está organizado de la siguiente manera:

* `/docs`: Documentación adicional, guías, lecturas complementarias.
* `/lectures`: Presentaciones y materiales de clase.
* `/assignments`: Enunciados de tareas y ejercicios propuestos.
* `/examples`: Ejemplos de código y proyectos pequeños para ilustrar conceptos.
* `/projects`: Directorios para los proyectos principales del curso (ej. Proyecto Final).
    * `/projects/project_name/src`: Código fuente del proyecto.
    * `/projects/project_name/docs`: Documentación específica del proyecto.
* `/resources`: Enlaces útiles, bibliografía.

## 🤝 Contribución

Se espera que los estudiantes contribuyan a sus respectivos proyectos y, cuando sea aplicable, colaboren en la mejora de los ejemplos o la documentación del curso.

Para contribuir:

1.  Haz un "fork" de este repositorio (si aplica para ejemplos o documentación).
2.  Crea una nueva rama (`git checkout -b feature/nombre-de-tu-rama`).
3.  Realiza tus cambios.
4.  Haz "commit" de tus cambios (`git commit -m 'feat: descripción breve de tu cambio'`).
5.  Sube tus cambios a tu "fork" (`git push origin feature/nombre-de-tu-rama`).
6.  Abre un "Pull Request" (PR) desde tu "fork" a la rama `main` de este repositorio.
